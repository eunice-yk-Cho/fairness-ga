FIX #1  [CRITICAL]  GA missing elitism — best individual lost every generation
────────────────────────────────────────────────────────────────────────────────

  File:     src/genetic_algorithm.py
  Method:   GeneticAlgorithm.run()

  PROBLEM:
    Every generation, the entire population was replaced unconditionally:

        # BEFORE (lines 116-118)
        if new_population:
            population = new_population
            pop_fitness = new_fitness

    The best individual from generation N was never carried over to
    generation N+1.  This is a well-known anti-pattern in GA design.
    Without elitism the GA can "forget" the best solution it ever found,
    causing the search to regress.

  FIX:
    Before building the new generation, the single fittest individual from the
    current population is copied into the next generation without re-evaluation.
    The inner loop now produces (POPULATION_SIZE - 1) offspring so the total
    evaluation budget is unchanged.

        # AFTER (lines 87-96)
        while eval_count < max_evals:
            elite_idx = int(np.argmax(pop_fitness))
            elite = population[elite_idx].copy()
            elite_fit = pop_fitness[elite_idx]

            new_population = [elite]
            new_fitness = [elite_fit]
            gen_scores = []

            for _ in range(self.config.POPULATION_SIZE - 1):
                ...

    The unconditional replacement at the bottom is now safe because the
    elite is always present:

        population = new_population
        pop_fitness = new_fitness

FIX #2  [CRITICAL]  best_curve tracked per-generation max, not running max
────────────────────────────────────────────────────────────────────────────────

  Files:    src/genetic_algorithm.py, src/random_search.py

  PROBLEM:
    A "convergence curve" must be monotonically non-decreasing (the best
    score found *so far*).  Both GA and Random Search recorded only the
    max within the current generation/chunk, so the curve could go DOWN.

        # BEFORE — genetic_algorithm.py (line 113)
        best_curve.append(float(np.max(gen_scores)))

        # BEFORE — random_search.py (line 18)
        best_curve.append(float(np.max(chunk)))

  FIX:
    Introduced a `running_best` variable that is updated with each new
    generation/chunk and never decreases.

        # AFTER — genetic_algorithm.py (lines 77-78, 119-122)
        running_best = float(np.max(pop_fitness))
        best_curve = [running_best]
        ...
            if gen_scores:
                gen_best = float(np.max(gen_scores))
                running_best = max(running_best, gen_best)
                best_curve.append(running_best)

        # AFTER — random_search.py (lines 9, 19-20)
        running_best = -np.inf
        ...
            running_best = max(running_best, float(np.max(chunk)))
            best_curve.append(running_best)

FIX #3  [HIGH]  Sampler produced continuous floats for categorical features
────────────────────────────────────────────────────────────────────────────────

  Files:    src/utils.py, src/preprocessing.py, src/genetic_algorithm.py,
            experiments/run_experiment.py

  PROBLEM:
    Features like workclass, education, marital-status (Adult), c_charge_degree
    (COMPAS), etc. are LabelEncoded to integers 0, 1, 2, ...  The sampler used
    np.random.uniform() for ALL features, producing values like 2.37 that the
    model never saw during training (out-of-distribution).

        # BEFORE — utils.py (lines 11-12)
        def sampler():
            return np.array([np.random.uniform(lo, hi) for lo, hi in feature_ranges])

    Similarly, mutation in the GA added Gaussian noise to every feature
    regardless of type.

  FIX:

    a) preprocessing.py now identifies and returns categorical column indices:

        # AFTER — preprocessing.py (lines 40-50)
        categorical_cols = set()
        for col in X.columns:
            if not pd.api.types.is_numeric_dtype(X[col]):
                le = LabelEncoder()
                X[col] = le.fit_transform(X[col].astype(str))
                encoders[col] = le
                categorical_cols.add(col)

        col_list = list(X.columns)
        sensitive_index = col_list.index(sensitive_attr)
        categorical_indices = frozenset(col_list.index(c) for c in categorical_cols)
        ...
        return X_train, X_test, y_train, y_test, sensitive_index, categorical_indices

    b) utils.py samples integers for categorical features:

        # AFTER — utils.py (lines 9-23)
        def make_sampler(feature_ranges, categorical_indices=frozenset()):
            def sampler():
                x = np.empty(len(feature_ranges))
                for i, (lo, hi) in enumerate(feature_ranges):
                    if i in categorical_indices:
                        x[i] = np.random.randint(int(lo), int(hi) + 1)
                    else:
                        x[i] = np.random.uniform(lo, hi)
                return x
            return sampler

    c) genetic_algorithm.py mutates categorical features by re-sampling an
       integer rather than adding Gaussian noise:

        # AFTER — genetic_algorithm.py (lines 34-46)
        def mutate(self, x):
            x = x.copy()
            if np.random.rand() < self.config.MUTATION_RATE:
                idx = np.random.randint(len(x))
                if idx in self.categorical_indices:
                    lo, hi = self.feature_ranges[idx]
                    x[idx] = np.random.randint(int(lo), int(hi) + 1)
                else:
                    x[idx] += np.random.normal(0, 0.1)
                    if self.feature_ranges is not None:
                        lo, hi = self.feature_ranges[idx]
                        x[idx] = np.clip(x[idx], lo, hi)
            return x

    d) run_experiment.py threads categorical_indices through to GA and sampler
       (lines 48-58, 60-72, 143-157, 168-174, 221-234).

FIX #4  [HIGH]  German Credit "unknown" sex treated as "male"
────────────────────────────────────────────────────────────────────────────────

  File:     scripts/download_benchmark_datasets.py
  Function: download_german()

  PROBLEM:
    personal_status codes not in the sex_map were filled with "unknown":

        # BEFORE (line 85)
        df["sex"] = df["personal_status"].map(sex_map).fillna("unknown")

    Since dropna() only drops NaN (not "unknown"), these rows survived.
    In _normalize_sensitive(), LabelEncoder ordered them alphabetically:
    "female"=0, "male"=1, "unknown"=2.  The binarisation
    (encoded > encoded.min()) mapped "unknown" to the same group as "male",
    silently corrupting the fairness analysis.

  FIX:
    Unmapped rows now stay as NaN and are removed by the existing dropna():

        # AFTER (line 85)
        df["sex"] = df["personal_status"].map(sex_map)

FIX #5  [LOW]  Dead code — unnecessary clip in crossover
────────────────────────────────────────────────────────────────────────────────

  File:     src/genetic_algorithm.py
  Method:   GeneticAlgorithm.crossover()

  PROBLEM:
    Uniform crossover copies each gene from one of two parents.  Both parents
    are already within feature_ranges, so the child is guaranteed to be in
    range.  The np.clip was dead code.

        # BEFORE (lines 27-34)

            child = np.where(mask, p1, p2)
            if self.feature_ranges is not None:
                lo = np.array([r[0] for r in self.feature_ranges])
                hi = np.array([r[1] for r in self.feature_ranges])
                child = np.clip(child, lo, hi)
            return child

  FIX:
    Removed the unnecessary clip:

        # AFTER (lines 29-32)
        def crossover(self, p1, p2):
            mask = np.random.rand(len(p1)) < 0.5
            child = np.where(mask, p1, p2)
            return child

FIX #13  [HIGH]  Visualization crash — GA/RS curve length mismatch
────────────────────────────────────────────────────────────────────────────────

  File:     experiments/visualization.py

  PROBLEM:
    After adding elitism (FIX #1), GA evaluates POPULATION_SIZE - 1 offspring
    per generation, while Random Search used chunk_size = POPULATION_SIZE.
    This caused GA and RS convergence curves to have different lengths.
    visualization.py used a single `steps` array derived from GA for both,
    crashing on the RS plot.

    NOTE: This was a regression introduced by FIX #1 (elitism).  The original
    code did not have this problem.

  FIX:
    a) run_experiment.py: RS chunk_size changed from POPULATION_SIZE to
       max(1, POPULATION_SIZE - 1) to match GA's per-generation eval count.

    b) visualization.py: Refactored convergence plotting into a helper
       function that creates independent x-axes for GA and RS curves,
       so any remaining length differences are handled gracefully.

FIX #16  [MEDIUM]  report.tex — DP/EO interpretation on biased samples
────────────────────────────────────────────────────────────────────────────────

  File:     report.tex  (Setup section + Discussion section)

  PROBLEM (NARRATIVE):
    DP and EO proxy are computed on GA/RS-generated samples, not real data.
    Because GA steers toward discriminatory regions, these are search-biased
    metrics.  The report presented them alongside ID count/mean without
    clarifying that they measure search effectiveness, not the model's actual
    fairness.

  FIX:
    a) Setup: DP metric description now states "computed over *generated*
       (not real-data) samples" and explains the implication.

    b) Discussion: Added paragraph after the DP variance discussion:

        "Readers should therefore interpret the DP and EO-proxy columns in
        Table 1 as measures of *search-biased fairness gap* — i.e. how
        large a disparity the search method can uncover — rather than as
        estimates of the model's demographic parity or equalized odds on
        the real data distribution."

FIX #20  [MEDIUM]  Sensitive attribute sampled as continuous float, not {0,1}
────────────────────────────────────────────────────────────────────────────────

  File:     src/preprocessing.py

  PROBLEM:
    The categorical-feature fix (FIX #3) correctly identified LabelEncoded
    columns, but the sensitive attribute was already converted to int {0,1}
    by _normalize_sensitive() BEFORE the LabelEncoder loop.  Because
    is_numeric_dtype(int) is True, the sensitive column was never added to
    categorical_cols, so the sampler used np.random.uniform(0, 1) — producing

    While this did not invalidate results (RandomForest splits handle it, and
    flip_sensitive always crosses the 0.5 boundary), it was an unintended gap
    in the categorical fix.

  FIX:
    Explicitly add sensitive_index to categorical_indices:

        # BEFORE
        categorical_indices = frozenset(
            col_list.index(c) for c in categorical_cols
        )

        # AFTER
        categorical_indices = frozenset(
            col_list.index(c) for c in categorical_cols

FIX #24  [MEDIUM]  Mutation σ=0.1 hardcoded — ineffective for wide-range features
────────────────────────────────────────────────────────────────────────────────

  File:     src/genetic_algorithm.py
  Method:   mutate()

  PROBLEM:
    Gaussian mutation used a fixed σ=0.1 for all continuous features.  For
    features like capital-gain (range [0, 99999]), this perturbation is
    0.0001% of the range — effectively a no-op.  Only features with range ≈1
    received meaningful mutation.

        # BEFORE
        x[idx] += np.random.normal(0, 0.1)

  FIX:
    σ is now 5% of the feature range, ensuring mutation is proportional to
    each feature's scale:

        # AFTER
        lo, hi = self.feature_ranges[idx]
        sigma = (hi - lo) * 0.05
        x[idx] += np.random.normal(0, max(sigma, 1e-9))
        x[idx] = np.clip(x[idx], lo, hi)
