Extension plan — things to do before final submission
====================================================

Core problem stays the same: AI Model Fairness Testing, GA vs Random Search.
These are improvements to the evaluation side only. Nothing changes in the
algorithm or the baseline.

1) Multiple datasets
   - Right now only Adult is tested.  That's not enough to say anything general.
   - Add COMPAS and German Credit so we can see if results hold across
     different data distributions.
   - Keep everything else identical (same classifier, same budget, same config).

2) Multiple fairness metrics
   - Individual discrimination alone is a narrow view.
   - Also compute demographic parity difference and equalized odds difference.
   - Run the statistical test separately for each metric.

3) Hyperparameter sensitivity
   - GA results could just be lucky parameter picks.
   - Vary population size (10, 20, 40) and mutation rate (0.05, 0.1, 0.2).
   - Keep total budget constant so comparison stays fair.

4) Convergence curves
   - Only showing final numbers doesn't prove the GA is actually *learning*.
   - Record best and mean fitness per generation and plot them.
   - If the curve is flat, GA is basically doing random search with extra steps.

5) Better related work section
   - Current section is too thin.  Need to mention search-based testing,
     fairness testing tools, and evolutionary optimization properly.
   - Use proper references (IEEE style).

What this should achieve:
- Stronger statistical evidence (more trials, more datasets, more metrics).
- Shows the algorithm is robust (sensitivity + convergence).
- Better academic positioning (expanded related work).
- Still one problem, one baseline, one tool — fully within project scope.
