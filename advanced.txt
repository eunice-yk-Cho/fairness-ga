Extension plan — status after implementation
=============================================

Core problem: AI Model Fairness Testing, GA vs Random Search.
All items below have been implemented. Nothing changes in the
baseline comparison design (same budget, same classifier, same seeds).

1) Multiple datasets                                            [DONE]
   - Adult, COMPAS, and German Credit are all supported.
   - Controlled via --datasets flag in run_experiment.py.
   - Results are saved per-dataset under experiments/results/<key>/.

2) Multiple fairness metrics                                    [DONE]
   - Individual discrimination count and mean.
   - Demographic parity difference (computed over generated samples;
     reflects search effectiveness, not the model's real-data DP).
   - Counterfactual prediction gap (EO proxy; conditions on
     counterfactual model predictions, not true labels).
   - Wilcoxon signed-rank test run separately for each metric.

3) Hyperparameter sensitivity                                   [DONE]
   - Population size {10, 20, 40} x mutation rate {0.05, 0.1, 0.2}.
   - Total budget kept constant so comparison stays fair.
   - Enabled via --with-sensitivity flag.

4) Convergence curves                                           [DONE]
   - Running-best and per-generation mean recorded.
   - Best curve is monotonically non-decreasing (running maximum).
   - Visualised with mean ± std across trials.

5) Better related work section                                  [DONE]
   - Covers fairness definitions, testing tools (Themis, Aequitas,
     white-box methods), and SBSE background.
   - Proper IEEE-style references.

6) Elitism                                                      [DONE]
   - Best individual is preserved across generations.
   - Inner loop generates POPULATION_SIZE - 1 offspring per generation.

7) Categorical feature handling                                 [DONE]
   - LabelEncoded columns are identified during preprocessing.
   - Sampler and mutation use integer sampling for those features.

8) Model accuracy logging                                       [DONE]
   - Train/test accuracy printed and saved in combined_results.json.
